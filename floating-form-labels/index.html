CSS only

Where CSS selectors alone apply down the DOM tree, the <input> on which events are based is placed before the <label> on which display actions are taken. As long as the <input> and the <label> are explicitely associated this presents no accessibility problems.

CSS & JS

Using Javascript allows the <label> and the <input> to be in the correct order in the DOM as scripting allows navigation up the DOM tree, events on the <input> influencing the <label> render by switching HTML classes.
